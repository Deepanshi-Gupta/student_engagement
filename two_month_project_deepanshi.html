Real-time classroom engagement monitor — 2-month M.Tech project design

Real-time multimodal student engagement & distress monitor
A classroom AI assistant that watches students via webcam and listens via mic in parallel — detecting attention, confusion, fatigue, and emotional distress — then uses Gemini to generate real-time alerts and adaptive suggestions for the teacher. Combines existing pre-trained models in 3 parallel CV streams fused into one LLM reasoning layer. No model training from scratch.

Why this fits your philosophy exactly

4 layers — all existing tools, you build the glue
Layer 1 — Vision (parallel streams)
Stream A: MediaPipe Face Mesh → 468 landmarks → head pose (yaw/pitch/roll) → attentive / distracted / away
Stream B: DeepFace / FER+ model → 7 emotion classes → confused / frustrated / bored / focused
Stream C: MediaPipe Eye landmarks → blink rate + gaze direction → fatigue / drowsy
All 3 run on the same webcam frame. Each emits a label + confidence score per second.
MediaPipe (Google, free)DeepFace (pip install)FER2013 (HuggingFace)
Layer 2 — Speech (parallel to vision)
Whisper → transcribes student speech in real time → keyword extraction (confusion markers: "huh?", "I don't understand", "what?")
Wav2Vec2 fine-tuned on RAVDESS → speech emotion (angry / distressed / calm / frustrated)
Both run async in a background thread. Output: [transcript snippet, speech_emotion, confusion_keywords].
openai/whisper-smallRAVDESS datasetHuggingFace pipeline
Layer 3 — Fusion (your original contribution)
Your weighted fusion function combines all 5 signals into one engagement score (0–100) and one distress flag (True/False).
Weights: head_pose=0.25, facial_emotion=0.30, eye_state=0.20, speech_emotion=0.15, confusion_keywords=0.10
Rolling 10-second window average smooths noise. This weighting schema + window = your thesis design decision.

Layer 4 — Reasoning + output (GenAI + FastAPI)
Every 30 seconds: engagement_vector → Gemini prompt → teacher alert ("3 students show confusion signals, consider pausing") or student nudge ("You seem distracted, take a breath and refocus").
FastAPI serves: /stream (live score), /alert (latest Gemini message), /report (session summary PDF).
Dashboard: Matplotlib real-time chart of engagement over time per student.

8 weeks — what you do each week
Week 1
Setup + MediaPipe head pose baseline
Install MediaPipe, DeepFace, SNNTorch, Tonic. Get webcam feed working. Implement head pose (yaw/pitch/roll) → attentive/distracted label. Test on yourself for 30 min. Repo structure, README started. End of week: 1 working CV stream.
Week 2
Add DeepFace emotion + eye blink streams
Run DeepFace on same frame as MediaPipe. Add eye blink rate counter from face mesh landmarks. Now 3 parallel streams on one frame. Measure FPS — if below 15, optimise: skip every 2nd frame for DeepFace. End of week: 3 parallel CV streams running.
Week 3
Add Whisper + Wav2Vec2 speech layer
Background thread: PyAudio captures mic → Whisper transcribes 5-second chunks → keyword search. Second thread: Wav2Vec2 speech emotion on same audio chunk. Test with your own voice saying confused/frustrated sentences. End of week: speech layer running async alongside CV.
Week 4
Fusion module + engagement score
Write the fusion.py module — weighted combination of 5 signals → engagement_score (0–100) + distress_flag. Rolling 10s window. Unit test: simulate edge cases (all signals distressed, all engaged, mixed). This is the week where your original contribution takes shape. Document weight choices in design doc.
Week 5
Gemini integration + alert generation
Every 30 seconds: format engagement_vector as structured prompt → Gemini → alert string. Implement 3 alert types: teacher alert, student nudge, session summary. Test: does Gemini produce useful, non-alarming alerts? Tune the prompt (3–4 iterations). End of week: full pipeline working end-to-end, ugly but functional.
Week 6
FastAPI + dashboard
Wrap everything in FastAPI. Endpoints: /stream, /alert, /report. Matplotlib chart: engagement score over time, per student. Postman-test all endpoints. Docker image. Deploy to GCP Cloud Run. End of week: someone else can run it with one Docker command.
Week 7
Ablation study (your thesis contribution)
Run the system with: (a) vision only, (b) speech only, (c) vision + speech. Record engagement scores and distress detection across 5 simulated sessions. Compare: does multimodal fusion outperform single-modal? This comparison IS your research result. Table 1 in your thesis.
Week 8
Thesis write-up + conference paper draft
5 chapters: Intro, Related Work, System Architecture, Experiments, Conclusion. 15–20 pages. The architecture diagram (your 4 layers) is Figure 1. The ablation table is Table 1. Submit to IEEE EDUCON or NCVPRIPG. Done.
